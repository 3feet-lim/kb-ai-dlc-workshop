# 테이블오더 서비스 요구사항

## Intent Analysis

- **User Request**: 테이블오더 서비스 구축 (디지털 주문 시스템)
- **Request Type**: New Project (Greenfield)
- **Scope**: System-wide (고객 UI, 관리자 UI, 백엔드 API, 데이터베이스)
- **Complexity**: Complex (멀티 매장, 실시간 통신, 인증, 세션 관리)

---

## Technology Stack Decisions

| Layer | Technology | Notes |
|-------|-----------|-------|
| Backend | Python (FastAPI) | REST API + SSE |
| Frontend (Customer) | Next.js (React/TypeScript) | 별도 앱 |
| Frontend (Admin) | Next.js (React/TypeScript) | 별도 앱 |
| Database | PostgreSQL | 관계형 DB |
| Deployment | Docker Compose | 로컬/온프레미스 |
| PBT Framework | Hypothesis (Python), fast-check (TypeScript) | PBT-09 준수 |

## Functional Requirements

### FR-01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 초기 설정 시 매장 식별자, 테이블 번호, 테이블 비밀번호 입력
- 로그인 정보 로컬 저장 후 자동 로그인
- 1회 로그인 성공 후 저장된 정보로 자동 인증

### FR-02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면으로 표시
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

### FR-03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

### FR-04: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- 주문 성공 시: 주문 번호 표시, 장바구니 비우기, 메뉴 화면 자동 리다이렉트 (5초 후)
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

### FR-05: 주문 내역 조회 (고객)
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- SSE 기반 주문 상태 실시간 업데이트
- 현재 테이블 세션 주문만 표시

### FR-06: 매장 인증 (관리자)
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지, 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱, 로그인 시도 제한
- 관리자 계정은 사전 등록 (DB 시딩 또는 CLI)

### FR-07: 실시간 주문 모니터링 (관리자)
- SSE 기반 주문 목록 실시간 업데이트
- 테이블별 카드 형태 그리드/대시보드 레이아웃
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중/준비중/완료)
- 신규 주문 시각적 강조, 2초 이내 표시
- 테이블별 필터링

### FR-08: 테이블 관리 (관리자)
- 테이블 태블릿 초기 설정 (번호, 비밀번호, 16시간 세션)
- 주문 삭제 (확인 팝업, 즉시 삭제, 총 주문액 재계산)
- 테이블 세션 종료 (이용 완료): 주문 내역을 과거 이력으로 이동, 주문 목록/총액 리셋
- 과거 주문 내역 조회: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링

### FR-09: 메뉴 관리 (관리자)
- 메뉴 CRUD (등록/조회/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지
- 이미지 파일 업로드 지원 (서버 로컬 스토리지)
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증

### FR-10: 멀티 매장 지원
- 여러 매장이 각각 독립적으로 운영
- 매장별 메뉴, 테이블, 주문 데이터 격리
- 매장 식별자 기반 데이터 분리

## Non-Functional Requirements

### NFR-01: 성능
- SSE 기반 실시간 통신: 주문 발생 후 2초 이내 관리자/고객 화면 반영
- API 응답 시간: 일반 요청 500ms 이내

### NFR-02: 사용성
- 터치 친화적 UI (최소 44x44px 터치 타겟)
- 직관적 시각적 계층 구조
- 반응형 레이아웃

### NFR-03: 데이터 무결성
- 주문 데이터 트랜잭션 처리
- 장바구니 로컬 저장 (새로고침 시 유지)
- 세션 기반 주문 격리

### NFR-04: 배포 및 운영
- Docker Compose 기반 로컬/온프레미스 배포
- 컨테이너화된 서비스 (백엔드, 프론트엔드 2개, DB)

### NFR-05: 테스트
- Property-Based Testing (PBT) 적용 (Hypothesis for Python, fast-check for TypeScript)
- PBT 규칙 전체 적용 (PBT-01 ~ PBT-10)

---

## Excluded Features (Constraints)

- 결제 처리 (카드, 현금, PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS, 2FA)
- 이미지 리사이징/최적화, CMS, 광고
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도, 번역)

## Glossary

- **테이블 세션**: 고객이 첫 주문 시작부터 이용 완료 처리까지의 기간. 종료 후 새 고객의 첫 주문 시 새 세션 시작.
- **MVP**: Minimum Viable Product
- **SSE**: Server-Sent Events
