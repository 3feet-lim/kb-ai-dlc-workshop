# User Stories

스토리 분류: Feature-Based | 세분화: Medium | 수락 기준: Checklist | 우선순위: High/Medium/Low

---

## 고객용 기능 (Customer Features)

### US-01: 테이블 태블릿 자동 로그인 (FR-01)
**As a** 고객, **I want to** 테이블 태블릿에서 별도 로그인 없이 자동으로 인증되기를, **So that** 즉시 주문을 시작할 수 있다.

**Priority**: High
**Persona**: 고객

**Acceptance Criteria**:
- [ ] 관리자가 초기 설정한 매장 ID, 테이블 번호, 비밀번호가 로컬에 저장된다
- [ ] 저장된 정보로 자동 로그인이 수행된다
- [ ] 자동 로그인 성공 시 메뉴 화면이 표시된다
- [ ] 자동 로그인 실패 시 초기 설정 화면이 표시된다

---

### US-02: 메뉴 조회 및 탐색 (FR-02)
**As a** 고객, **I want to** 카테고리별로 메뉴를 탐색하기를, **So that** 원하는 메뉴를 쉽게 찾을 수 있다.

**Priority**: High
**Persona**: 고객

**Acceptance Criteria**:
- [ ] 메뉴 화면이 기본 화면으로 표시된다
- [ ] 카테고리별로 메뉴가 분류되어 표시된다
- [ ] 각 메뉴에 메뉴명, 가격, 설명, 이미지가 표시된다
- [ ] 카테고리 간 빠른 이동이 가능하다
- [ ] 카드 형태 레이아웃으로 표시된다
- [ ] 터치 타겟이 최소 44x44px이다

---

### US-03: 장바구니에 메뉴 추가 (FR-03)
**As a** 고객, **I want to** 메뉴를 장바구니에 추가하기를, **So that** 주문 전에 선택한 메뉴를 관리할 수 있다.

**Priority**: High
**Persona**: 고객

**Acceptance Criteria**:
- [ ] 메뉴를 장바구니에 추가할 수 있다
- [ ] 장바구니에서 메뉴를 삭제할 수 있다
- [ ] 수량을 증가/감소할 수 있다
- [ ] 총 금액이 실시간으로 계산된다
- [ ] 장바구니를 비울 수 있다
- [ ] 페이지 새로고침 시에도 장바구니가 유지된다

---

### US-04: 주문 생성 (FR-04)
**As a** 고객, **I want to** 장바구니의 메뉴를 주문으로 확정하기를, **So that** 매장에 주문이 전달된다.

**Priority**: High
**Persona**: 고객

**Acceptance Criteria**:
- [ ] 주문 확정 전 주문 내역을 최종 확인할 수 있다
- [ ] 주문 확정 버튼을 누르면 주문이 생성된다
- [ ] 주문 성공 시 주문 번호가 표시된다
- [ ] 주문 성공 시 장바구니가 자동으로 비워진다
- [ ] 주문 성공 후 5초 뒤 메뉴 화면으로 자동 리다이렉트된다
- [ ] 주문 실패 시 에러 메시지가 표시되고 장바구니가 유지된다
- [ ] 주문 정보에 매장 ID, 테이블 ID, 메뉴 목록, 총 금액, 세션 ID가 포함된다

---

### US-05: 주문 내역 조회 (FR-05)
**As a** 고객, **I want to** 현재 테이블의 주문 내역을 확인하기를, **So that** 주문 상태를 파악할 수 있다.

**Priority**: High
**Persona**: 고객

**Acceptance Criteria**:
- [ ] 주문이 시간 순으로 정렬되어 표시된다
- [ ] 각 주문에 주문 번호, 시각, 메뉴/수량, 금액, 상태가 표시된다
- [ ] 주문 상태(대기중/준비중/완료)가 SSE로 실시간 업데이트된다
- [ ] 현재 테이블 세션의 주문만 표시된다
- [ ] 이용 완료 처리된 주문은 표시되지 않는다

---

## 관리자용 기능 (Admin Features)

### US-06: 매장 관리자 로그인 (FR-06)
**As a** 관리자, **I want to** 매장 관리 시스템에 로그인하기를, **So that** 매장을 관리할 수 있다.

**Priority**: High
**Persona**: 관리자

**Acceptance Criteria**:
- [ ] 매장 식별자, 사용자명, 비밀번호로 로그인할 수 있다
- [ ] JWT 토큰 기반 인증이 적용된다
- [ ] 세션이 16시간 유지된다
- [ ] 브라우저 새로고침 시 세션이 유지된다
- [ ] 16시간 후 자동 로그아웃된다
- [ ] 비밀번호가 bcrypt로 해싱되어 저장된다
- [ ] 로그인 시도 횟수가 제한된다

---

### US-07: 실시간 주문 모니터링 (FR-07)
**As a** 관리자, **I want to** 들어오는 주문을 실시간으로 확인하기를, **So that** 주문을 빠르게 처리할 수 있다.

**Priority**: High
**Persona**: 관리자

**Acceptance Criteria**:
- [ ] SSE 기반으로 주문 목록이 실시간 업데이트된다
- [ ] 테이블별 카드 형태 그리드 레이아웃으로 표시된다
- [ ] 각 테이블 카드에 총 주문액과 최신 주문 미리보기가 표시된다
- [ ] 주문 카드 클릭 시 전체 메뉴 목록 상세가 표시된다
- [ ] 주문 상태를 대기중/준비중/완료로 변경할 수 있다
- [ ] 신규 주문이 시각적으로 강조된다
- [ ] 주문 발생 후 2초 이내에 화면에 표시된다
- [ ] 테이블별 필터링이 가능하다

---

### US-08: 테이블 초기 설정 (FR-08)
**As a** 관리자, **I want to** 테이블 태블릿을 초기 설정하기를, **So that** 고객이 해당 테이블에서 주문할 수 있다.

**Priority**: High
**Persona**: 관리자

**Acceptance Criteria**:
- [ ] 테이블 번호와 비밀번호를 설정할 수 있다
- [ ] 16시간 세션이 생성된다
- [ ] 설정 정보가 저장되고 자동 로그인이 활성화된다

---

### US-09: 주문 삭제 (FR-08)
**As a** 관리자, **I want to** 특정 주문을 삭제하기를, **So that** 잘못된 주문을 정정할 수 있다.

**Priority**: Medium
**Persona**: 관리자

**Acceptance Criteria**:
- [ ] 특정 주문에 삭제 버튼이 표시된다
- [ ] 삭제 전 확인 팝업이 표시된다
- [ ] 확인 시 주문이 즉시 삭제된다
- [ ] 테이블 총 주문액이 재계산된다
- [ ] 성공/실패 피드백이 표시된다

---

### US-10: 테이블 이용 완료 처리 (FR-08)
**As a** 관리자, **I want to** 테이블 이용 완료를 처리하기를, **So that** 새 고객이 이전 주문 없이 시작할 수 있다.

**Priority**: High
**Persona**: 관리자

**Acceptance Criteria**:
- [ ] 테이블 이용 완료 버튼이 있다
- [ ] 확인 팝업이 표시된다
- [ ] 해당 세션의 주문 내역이 과거 이력으로 이동된다
- [ ] 테이블 현재 주문 목록과 총 주문액이 0으로 리셋된다
- [ ] 성공/실패 피드백이 표시된다

---

### US-11: 과거 주문 내역 조회 (FR-08)
**As a** 관리자, **I want to** 테이블의 과거 주문 내역을 조회하기를, **So that** 이전 매출을 확인할 수 있다.

**Priority**: Medium
**Persona**: 관리자

**Acceptance Criteria**:
- [ ] "과거 내역" 버튼으로 접근할 수 있다
- [ ] 테이블별 과거 주문 목록이 시간 역순으로 표시된다
- [ ] 각 주문에 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다
- [ ] 날짜 필터링이 가능하다
- [ ] "닫기" 버튼으로 대시보드로 복귀할 수 있다

---

### US-12: 메뉴 관리 (FR-09)
**As a** 관리자, **I want to** 메뉴를 등록/수정/삭제하기를, **So that** 매장 메뉴를 최신 상태로 유지할 수 있다.

**Priority**: High
**Persona**: 관리자

**Acceptance Criteria**:
- [ ] 카테고리별로 메뉴를 조회할 수 있다
- [ ] 메뉴를 등록할 수 있다 (메뉴명, 가격, 설명, 카테고리, 이미지)
- [ ] 이미지 파일을 업로드할 수 있다
- [ ] 메뉴를 수정할 수 있다
- [ ] 메뉴를 삭제할 수 있다
- [ ] 메뉴 노출 순서를 조정할 수 있다
- [ ] 필수 필드 검증이 수행된다
- [ ] 가격 범위 검증이 수행된다

---

## 멀티 매장 (Multi-Store)

### US-13: 멀티 매장 데이터 격리 (FR-10)
**As a** 관리자, **I want to** 내 매장의 데이터가 다른 매장과 격리되기를, **So that** 독립적으로 매장을 운영할 수 있다.

**Priority**: High
**Persona**: 관리자

**Acceptance Criteria**:
- [ ] 매장별로 메뉴 데이터가 독립적으로 관리된다
- [ ] 매장별로 테이블 데이터가 독립적으로 관리된다
- [ ] 매장별로 주문 데이터가 독립적으로 관리된다
- [ ] 매장 식별자 기반으로 데이터가 분리된다
