# Business Rules - Backend API

## BR-01: 인증 규칙

### BR-01-1: 관리자 로그인
- 매장 식별자 + 사용자명 + 비밀번호로 인증
- 비밀번호는 bcrypt로 검증
- 성공 시 JWT 토큰 발급 (만료: 16시간)
- 실패 시 failed_login_attempts 증가
- 5회 연속 실패 시 30분 잠금 (locked_until 설정)

### BR-01-2: 테이블 태블릿 로그인
- 매장 식별자 + 테이블 번호 + 비밀번호로 인증
- 성공 시 세션 토큰 발급 (만료: 16시간)
- 토큰에 store_id, table_id 포함

## BR-02: 주문 상태 전이 규칙
- 단방향만 허용: PENDING → PREPARING → COMPLETED
- 허용되지 않는 전이 시 에러 반환
- 상태 변경 시 SSE 이벤트 발행

```
PENDING ──→ PREPARING ──→ COMPLETED
  (대기중)     (준비중)      (완료)
```

## BR-03: 테이블 세션 규칙

### BR-03-1: 세션 시작
- 첫 주문 생성 시 자동으로 세션 시작
- 해당 테이블에 활성 세션이 없으면 새 세션 생성
- 활성 세션이 있으면 기존 세션에 주문 추가

### BR-03-2: 세션 종료 (이용 완료)
- 관리자만 이용 완료 처리 가능
- 해당 세션의 모든 주문을 OrderHistory로 이동
- 세션 is_active = false, ended_at 설정
- 테이블 주문 목록/총액 리셋

## BR-04: 주문 번호 생성 규칙
- 매장별 일련번호, 매일 리셋
- 형식: "#001", "#002", ...
- DailyOrderCounter 테이블에서 해당 매장+날짜의 counter를 atomic하게 증가
- 날짜가 바뀌면 새 카운터 레코드 생성 (counter=1부터)

## BR-05: 주문 생성 규칙
- 장바구니 항목이 1개 이상이어야 함
- 각 항목의 메뉴가 해당 매장에 존재하고 is_available이어야 함
- 주문 시점의 메뉴명, 단가를 스냅샷으로 저장
- total_amount = sum(quantity * unit_price)
- 주문 생성 시 SSE 이벤트 발행

## BR-06: 주문 삭제 규칙
- 관리자만 삭제 가능
- 삭제 시 해당 주문의 OrderItem도 함께 삭제
- 삭제 후 SSE 이벤트 발행

## BR-07: 메뉴 관리 규칙
- 메뉴명: 필수, 최대 100자
- 가격: 필수, 0 이상
- 카테고리: 필수, 해당 매장의 카테고리여야 함
- 이미지: 선택, 파일 업로드 시 URL 저장
- 노출 순서: sort_order 값으로 정렬

## BR-08: 카테고리 관리 규칙
- 관리자가 CRUD 가능
- 카테고리명: 필수, 매장 내 중복 불가
- 카테고리 삭제 시 해당 카테고리의 메뉴가 있으면 삭제 불가

## BR-09: 멀티 매장 데이터 격리
- 모든 API 요청에서 store_id 기반 데이터 필터링
- Repository 레이어에서 store_id 조건 강제 적용
- 다른 매장의 데이터 접근 시 403 에러

## BR-10: 파일 업로드 규칙
- 허용 확장자: jpg, jpeg, png, gif, webp
- 파일명: UUID로 변환하여 저장
- 저장 경로: /uploads/{store_id}/{filename}
